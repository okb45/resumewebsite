Variable Rate Fertilizer

Variable-rate fertilizer application (VRA) improves nutrient efficiency and field uniformity, yet commercial VRA spreaders commonly exceed $30,000, limiting adoption. This project demonstrates a retrofit variable-rate controller for a pre-existing fertilizer spreader, targeting low cost and ease of installation. The controller couples a microcontroller with a linear or rotary actuator to modulate gate opening (or conveyor/feed rate) in real time based on setpoints.

To provide feedback and mapping, the system integrates with myEV, a free web application developed for the project. myEV acts as a data collector and visualization tool, logging time-stamped operating data (GPS position, commanded fertilizer rate, actuator state) and automatically producing a field application map. By pairing low-cost hardware with an open-access digital platform, the system delivers the core value of VRA — rate control, traceability, and decision support — without the premium price tag of commercial equipment.

Proof of Conecept

We retrofitted Vicon fertilizer spreader with a linear actuator to adjust the gate opening in real time. A Raspberry Pi runs the control loop and communicates with the actuator via a motor driver. A compact ADC measures actuator position feedback, while an RTK GPS provides centimeter-level positioning.

We validated the system in a vineyard block using a urea-based fertilizer. The Raspberry Pi loaded a vineyard shapefile with per-polygon target rates. As the tractor traversed the rows, the controller adjusted the gate opening to match the desired application rate for each block.

Easy to use UI interfacing with myEV

To complement the hardware system, a PyQt-based application was developed to serve as the operator interface on a rugged touch screen monitor. The app allows users to configure system parameters, calibrate actuator positions, and initiate fertilizer application runs with a simple, modern GUI. It also provided real-time status feedback from the Raspberry Pi, improving usability and reducing setup time in the field.

Additionally, a custom PCB design was created to consolidate the wiring and signal routing between the Raspberry Pi, ADC, and actuator. The PCB replaced the temporary breadboard wiring used during early prototyping, resulting in a more robust and reliable system suitable for extended field trials. This design step moved the project closer to a practical retrofit solution that could be reproduced or scaled.
